# 04 B2C 챗봇 — UI 명세

## 1. 사이드바

| 요소 | 명세 |
|---|---|
| 페르소나 | 4택. 저학년(1~3)이면 "마이페이스 / 기본" disable |
| 학년 | 1~6 라디오 |
| 도착지 | 관악구 초등학교 드롭다운 |

---

## 2. 지도 — 경로 색상

엣지 단위 색상 (`core/road_risk.py: risk_band()` 기준):

| risk 구간 | 라벨 | 색 |
|---|---|---|
| `≤ 0.33` (RISK_SAFE) | 안전 | 초록 `#1D9E75` |
| `(0.33, 0.66]` (RISK_HIGH) | 주의 | 노랑 `#FFE066` |
| `(0.66, 0.85]` (RISK_SEVERE) | 위험 | 주황 `#FF9933` |
| `> 0.85` | 매우 위험 | 빨강 `#E63946` |

- 스쿨존 폴리곤: 반투명 노랑 오버레이 (시각 전용, 라우팅 로직과 무관)
- 지도 라이브러리: leaflet 또는 mapbox-gl

---

## 3. 챗 영역

- 자연어 입력창
- 응답 영역: 안내문(사실/표현 분리 파이프라인 출력) + 적용된 nudge 1줄 + (필요 시) warning 문구

warning 문구 예시: `"이 시간 안엔 충분히 안전한 길이 없어요"`

---

## 4. 빌드 순서

각 단계는 독립적으로 데모 가능하다.

1. 그래프 + OD 스냅 + Lagrangian Dijkstra 루프
2. 색상 경로 + 스쿨존 오버레이 지도 시각화
3. 규칙 기반 요인 식별기
4. LLM constrained 안내문 + 템플릿 폴백
5. 자연어 nudge 파이프라인

---

## 5. 구현 범위 밖

| 항목 | 설명 |
|---|---|
| 요인-한정 트리거 | 특정 위험 요인만 회피하는 세밀 제어 |
| nudge 강도 3단계 | "조금 / 꽤 / 너무" 같은 연속 강도 |
| 하굣길 | 학교 → 집 방향 (baseline 구조 다름) |
| 학부모 설문 / 문헌 인용 | UI 내 근거 자료 표시 |
